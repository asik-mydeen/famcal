## Gamification Model
- **Points:** `task.points_value` ├Ч priority multiplier (high=2x, medium=1x, low=0.5x)
- **Levels:** `Math.floor(points / 100) + 1` тЖТ Beginner to Supreme
- **Rewards:** Claimable items, deducts `points_cost` from member balance
- **Streaks:** `streak_days` tracked, incremented on daily completion
- **Recurring:** `completed: false` always, `completed_at` = last completion date

## Google Calendar Server-Side Sync

**Two-layer sync:**
1. **Server cron** (`/api/google-sync.js`) тАФ every 15 min, uses refresh tokens, no browser needed
2. **Client implicit** (`lib/googleCalendar.js`) тАФ 1hr tokens, disabled when server sync active

**Connection flow:** `connectMemberCalendar` тЖТ auth code flow тЖТ `/api/google-token` тЖТ refresh token stored in `family_members.google_refresh_token`

**Dedup logic (prevents duplicates):**
1. Primary: match by `google_event_id`
2. Fallback: match by `title + start_time` (links orphaned manual events)
3. DB unique index on `google_event_id` catches race conditions
4. Client auto-sync disabled when all members have `has_server_sync`

**Connection status (4 states):** connected (local token) | synced-via-server (refresh token) | needs-reconnect | not-connected

## Realtime Sync (BROADCAST CHANNELS тАФ WORKING)

- **Channel:** `family:{familyId}` broadcast (NOT postgres_changes)
- **Flow:** Client writes тЖТ Supabase INSERT тЖТ 1.5s delay тЖТ broadcast тЖТ other clients refetch
- **Dashboard/kiosk:** Subscribes to broadcast, refetches via `/api/dashboard`

## Known Gotchas
1. **`--openssl-legacy-provider`** required for CRA 5.0.1 with newer Node.js
2. **`--legacy-peer-deps`** required for npm install
3. **Event IDs** тАФ `crypto.randomUUID()` for all events (NOT `evt-xxx` temp IDs)
4. **Task IDs** тАФ still use `task-${Date.now()}` pattern, skip Supabase persist for temp IDs
5. **Theme engine complete** тАФ 4 presets, all colors centralized, DB sync, real-time switching
6. **No DashboardLayout/Sidenav** тАФ removed from template. Use PageShell + PageTransition
7. **Gold text contrast** тАФ `#FDCB6E` needs dark text (`#1a1a1a`)
8. **Absolute imports only** тАФ CRA configured with `src/` base path
9. **Icon fontSize** тАФ Always use `!important` (MUI override issue)
10. **CRA + useRef** тАФ refs inside useCallback break in prod тЖТ use module-level vars
11. **Google popups** тАФ `requestAccessToken(silentOnly=true)` = cache-only, NEVER GIS
12. **Dashboard contextValue** тАФ Don't use useMemo тАФ prevents re-renders
13. **SUPABASE_SERVICE_KEY required** тАФ anon key blocked by RLS for server writes
14. **Server sync dedup** тАФ checks ALL events (any source) + title+start fallback
15. **Client auto-sync disabled** тАФ when all members have `has_server_sync=true`
16. **Voice mode** тАФ Nova 2 Sonic (primary, `REACT_APP_NOVA_PROXY_URL`) тЖТ SpeechRecognition (Chrome) тЖТ Whisper (Firefox)
17. **Font legibility** тАФ Decorative fonts get size boost via `FONT_ADJUSTMENTS`
18. **Recurring tasks** тАФ `completed: false` always, `completed_at` = last done date
19. **memberFromDb** тАФ derives `has_server_sync` from `google_refresh_token`, strips token from client
20. **Nova turn_detection** тАФ only `threshold` field supported (not `silence_duration_ms` or other fields)
21. **ScriptProcessorNode** тАФ buffer size must be power-of-2 (use 4096, NOT 2400)
22. **Node.js heap** тАФ CRA build needs `--max-old-space-size=4096` (OOM at 19+ features)
23. **Service worker disabled** тАФ CRA 5 requires PWA template for service worker; standard template breaks it
24. **MoodBoard** тАФ redesigned as gradient icon pills (not raw emojis)
25. **Tauri macOSPrivateApi** тАФ MUST be `true` in tauri.conf.json + feature in Cargo.toml for getUserMedia in WKWebView
26. **Tauri kiosk entry** тАФ `/kiosk` тЖТ `/d/:slug/*` (Dashboard). Bypasses Google OAuth, uses slug+token auth
27. **Dashboard feature parity** тАФ Dashboard shell now has screensaver, voice, briefing, routines, emergency (same as App.js)
28. **CORS origin-restricted** тАФ API functions check `req.headers.origin` against allowlist (no wildcard)
29. **React.lazy code splitting** тАФ All page imports lazy-loaded in both App.js and Dashboard
30. **Tauri plugins** тАФ single-instance, autostart, process, window-state, global-shortcut, updater, opener
31. **PhotoFrame Ken Burns** тАФ Uses fixed KENBURNS_PRESETS (not random inline) to prevent CSS class accumulation
