## Gamification Model
- **Points:** `task.points_value` × priority multiplier (high=2x, medium=1x, low=0.5x)
- **Levels:** `Math.floor(points / 100) + 1` → Beginner to Supreme
- **Rewards:** Claimable items, deducts `points_cost` from member balance
- **Streaks:** `streak_days` tracked, incremented on daily completion
- **Recurring:** `completed: false` always, `completed_at` = last completion date

## Google Calendar Server-Side Sync

**Two-layer sync:**
1. **Server cron** (`/api/google-sync.js`) — every 15 min, uses refresh tokens, no browser needed
2. **Client implicit** (`lib/googleCalendar.js`) — 1hr tokens, disabled when server sync active

**Connection flow:** `connectMemberCalendar` → auth code flow → `/api/google-token` → refresh token stored in `family_members.google_refresh_token`

**Dedup logic (prevents duplicates):**
1. Primary: match by `google_event_id`
2. Fallback: match by `title + start_time` (links orphaned manual events)
3. DB unique index on `google_event_id` catches race conditions
4. Client auto-sync disabled when all members have `has_server_sync`

**Connection status (4 states):** connected (local token) | synced-via-server (refresh token) | needs-reconnect | not-connected

## Realtime Sync (BROADCAST CHANNELS — WORKING)

- **Channel:** `family:{familyId}` broadcast (NOT postgres_changes)
- **Flow:** Client writes → Supabase INSERT → 1.5s delay → broadcast → other clients refetch
- **Dashboard/kiosk:** Subscribes to broadcast, refetches via `/api/dashboard`
- **Fallback:** 15s polling if WebSocket fails
- **Fixed issues:** temp ID errors (→ crypto.randomUUID), ref closures (→ module vars), write cooldowns (→ removed)

## Known Gotchas
1. **`--openssl-legacy-provider`** required for CRA 5.0.1 with newer Node.js
2. **`--legacy-peer-deps`** required for npm install
3. **Event IDs** — `crypto.randomUUID()` for all events (NOT `evt-xxx` temp IDs)
4. **Task IDs** — still use `task-${Date.now()}` pattern, skip Supabase persist for temp IDs
5. **Theme engine complete** — 4 presets, all colors centralized, DB sync, real-time switching
6. **No DashboardLayout/Sidenav** — removed from template. Use PageShell + PageTransition
7. **Gold text contrast** — `#FDCB6E` needs dark text (`#1a1a1a`)
8. **Absolute imports only** — CRA configured with `src/` base path
9. **Icon fontSize** — Always use `!important` (MUI override issue)
10. **CRA + useRef** — refs inside useCallback break in prod → use module-level vars
11. **Google popups** — `requestAccessToken(silentOnly=true)` = cache-only, NEVER GIS
12. **Dashboard contextValue** — Don't use useMemo — prevents re-renders
13. **SUPABASE_SERVICE_KEY required** — anon key blocked by RLS for server writes
14. **Server sync dedup** — checks ALL events (any source) + title+start fallback
15. **Client auto-sync disabled** — when all members have `has_server_sync=true`
16. **Voice mode** — native SpeechRecognition (Chrome) + Whisper API fallback (Firefox)
17. **Font legibility** — Decorative fonts get size boost via `FONT_ADJUSTMENTS`
18. **Recurring tasks** — `completed: false` always, `completed_at` = last done date
19. **memberFromDb** — derives `has_server_sync` from `google_refresh_token`, strips token from client
